🔄 How It Works - Step by Step

Step 1: Upload

User drags/drops PDF file on the upload page. File stored in React state.

Step 2: Extract Text

Backend uses PyMuPDF to extract text from all pages of the PDF document.

Step 3: Chunk & Summarize

Text split into 700-word chunks. Each chunk summarized by BART AI model (Facebook).

Step 4: Generate Audio

Summary sent to ElevenLabs API in 200-word chunks. Audio files stitched together.

Step 5: Display Results

User receives text summary + audio player. Can read or listen to content.

💻 Technical Implementation

Backend Flow (Flask)

1. Receive PDF file → Save temporarily
2. Extract text using PyMuPDF
3. Split into 700-word chunks (token limit)
4. Pass each chunk to BART summarizer
5. Combine all summaries
6. Generate audio via ElevenLabs API
7. Return JSON: {summary: "...", audio_url: "/audio/file.mp3"}

Why Chunking?

AI models have token limits (~1024 tokens). Large PDFs must be split to avoid errors.

Audio Stitching

ElevenLabs API limited to 200 words. We generate multiple chunks and merge them using pydub.
